# MicroPython 现在要求每个拉取请求都需披露AI相关信息

MicroPython在其贡献者指南中新增了[生成式人工智能政策](https://github.com/micropython/micropython/wiki/ContributorGuidelines#generative-ai-policy)，而 [PR #18842](https://github.com/micropython/micropython/pull/18842) 则在GitHub的PR模板中添加了相应的声明。现在，每位贡献者都需要选择其中一项：“我没有使用生成式人工智能工具”或“我使用了这些工具，但代码已经过人工检查”

发起该PR的安格斯·格拉顿（Angus Gratton）表示：“在人工智能的辅助下编写嵌入式代码很容易，然而，如果没有人类的密切参与，生成式人工智能要生成高质量的嵌入式代码并不容易。结果呢？这些PR初看起来很棒，但在审查过程中却耗费了维护者的大量时间。”

他们并没有完全效仿 [Zig](https://github.com/micropython/micropython/pull/18842) 的做法，彻底禁止人工智能。格拉顿（Gratton）表示，他希望他们不必这么做，因为每一个草率的AI贡献背后，都伴随着一个人类深思熟虑的贡献。他们更希望人们能公开这些贡献，而不是将其隐藏起来。
